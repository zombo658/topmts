# Полная автоматизация: серверная отправка отчётов

> ⚠️ **Проверено: из облака GitHub этот вариант НЕ работает.** Портал
> `inventory.ural.mts.ru` блокирует зарубежные IP — серверы GitHub (США)
> до него не достучались (соединение отпадает по таймауту). Поэтому запуск
> по расписанию в GitHub отключён.
>
> Этот скрипт годится **только при запуске с машины внутри России** (свой
> VPS в РФ или офисный ПК на рабочей сети) — см. раздел «Если портал не
> открывается из облака» ниже. Для большинства случаев проще и надёжнее
> **Android-приложение** (`../android/`): телефон уже в нужной сети и шлёт
> от вашего имени.


Отчёты отправляются **из облака GitHub Actions** — телефоны и компьютеры
сотрудников не участвуют вообще. Работает одинаково для Android и iPhone,
даже если телефон выключен. Каждый день в назначенное время робот:

1. заходит на портал `inventory.ural.mts.ru` под логином каждого сотрудника;
2. собирает показатели со страницы `agent_day.php`
   (колонка «Результат оказания услуги», та же расшифровка, что в расширении);
3. отправляет отчёт каждого в рабочую беседу ВК через официальный
   Bot API — сообщением вида «Отчёт — Иван Иванов».

## Разовая настройка (делает один человек)

### 1. Бот-сообщество ВК

1. Создайте сообщество ВК (тип «Группа», название например «Отчёты отдела»).
2. В сообществе: **Управление → Настройки → Работа с API → Ключи доступа** →
   «Создать ключ» с правом **«Сообщения сообщества»**. Скопируйте ключ —
   это `VK_TOKEN`.
3. Там же в **Настройки → Сообщения** включите «Сообщения сообщества»,
   а в **Сообщения → Настройки для бота** включите «Возможности ботов» и
   разрешите добавление в беседы.
4. Добавьте сообщество в рабочую беседу (в беседе: «Добавить участника» →
   имя сообщества). В настройках участника-бота дайте «Доступ к переписке».

### 2. Секреты в GitHub

В репозитории: **Settings → Secrets and variables → Actions → New repository
secret**. Создайте три секрета:

| Имя | Значение |
|---|---|
| `VK_TOKEN` | ключ доступа сообщества из шага 1 |
| `VK_PEER` | id беседы: `2000000066` или `c66` |
| `EMPLOYEES` | список сотрудников в формате JSON, см. ниже |

Формат `EMPLOYEES` (логины/пароли — от портала МТС):

```json
[
  {"name": "Иван Иванов", "login": "ivanov", "password": "секрет1", "calls": "0"},
  {"name": "Пётр Петров", "login": "petrov", "password": "секрет2", "calls": "0"}
]
```

`calls` — количество звонков, которое подставится в отчёт (портал его
не показывает). Необязательное, по умолчанию 0.

Секреты видны только роботу — в коде и логах они не появляются.
Никогда не записывайте пароли прямо в файлы репозитория.

### 3. Время и дни отправки

- Время задаётся в `.github/workflows/report.yml` в строке `cron` **в UTC**:
  `0 13 * * *` = 18:00 по Екатеринбургу (UTC+5). Местное время минус 5 часов.
- Дни недели можно ограничить переменной `DAYS` (Settings → Secrets and
  variables → Actions → **Variables**): например `1,2,3,4,5` — только будни
  (0 — воскресенье). Без переменной — каждый день.
- Часовой пояс: переменная `TZ_OFFSET` (по умолчанию `5` — Урал).

## Проверка

Вкладка **Actions** репозитория → «Отправка отчётов в ВК» → **Run workflow**.
Через полминуты в беседе появятся отчёты, а в логе запуска — по строке
«✓ Имя: отчёт отправлен» на каждого сотрудника (или текст ошибки).

## Если портал не открывается из облака

Серверы GitHub находятся за границей — если портал МТС пускает только из
российских сетей, запуск упадёт с ошибкой соединения. Тогда тот же скрипт
можно запускать с любого компьютера/сервера в нужной сети:

```bash
EMPLOYEES='[{"name":"Иван","login":"...","password":"..."}]' \
VK_TOKEN='...' VK_PEER='c66' node automation/send-report.js
```

и поставить в планировщик (cron/Планировщик задач). На Android это же
работает в Termux.

## Ограничения

- Сообщения приходят **от имени бота-сообщества** с подписью, чей это
  отчёт, — не от личного аккаунта сотрудника. Это единственный способ,
  разрешённый правилами ВК.
- «Количество звонков» портал не показывает, поэтому в облачном режиме оно
  берётся из настройки `calls` сотрудника. Чтобы менять его день ко дню,
  проще отправлять отчёт с компьютера расширением или с телефона скриптом
  Tampermonkey — там число спрашивается перед отправкой.
