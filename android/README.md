# Отчёт МТС → ВК — приложение для Android

Одно приложение вместо связки «расширение + Tampermonkey + MacroDroid».
Внутри: экран настроек, вход в портал и в ВК (один раз), отправка отчёта
в чат ВК **от вашего имени** по расписанию. Всё работает через встроенный
браузер (WebView), никакого бота и никаких токенов.

## Как получить APK

APK собирается автоматически в GitHub Actions — компилировать у себя ничего
не нужно.

1. Откройте репозиторий на GitHub → вкладка **Actions** → workflow
   **«Сборка Android APK»** → **Run workflow** (или он запустится сам после
   изменений в папке `android/`).
2. Дождитесь зелёной галочки, откройте запуск и внизу в разделе
   **Artifacts** скачайте `otchet-mts-vk-apk`. Внутри — `app-debug.apk`.
3. Перекиньте файл на телефон и откройте его. Разрешите установку из этого
   источника («Неизвестные источники» / «Установка неизвестных приложений»).

Это debug-сборка — она устанавливается напрямую, без Google Play.

## Первый запуск (один раз)

1. Откройте приложение. Оно попросит разрешить **уведомления** — разрешите.
2. Нажмите **«Войти в портал МТС»** → войдите как обычно. Закройте вход
   кнопкой «назад» — сессия сохранится.
3. Нажмите **«Войти в ВК»** → войдите в свой аккаунт. «Назад».
4. Заполните: **чат** (значение `sel=…` из адреса, например `c66`),
   **домен ВК** (`vk.ru` или `vk.com`), **звонки**, **время** (по умолчанию
   22:00), **дни недели**, при желании — **шаблон**.
5. Поставьте галочку **«Автоотправка включена»** и нажмите **«Сохранить»**.
   Приложение попросит разрешить **«Точные будильники»** — разрешите
   (без этого отправка может опаздывать).
6. Нажмите **«Отправить сейчас»** для проверки — через уведомление увидите
   результат.

## Как это работает в назначенное время

Будильник будит приложение → скрытый WebView открывает портал, собирает
отчёт (колонка «Результат оказания услуги», расшифровка, сумма ДМХ, дата,
звонки, пустое = 0) → открывает ваш чат ВК → вставляет текст и жмёт
«отправить». Итог приходит уведомлением.

## Что важно

- **Телефон должен быть включён и в сети** к назначенному времени.
- Иногда ВК или портал просят перелогиниться — тогда снова нажмите
  соответствующую кнопку входа.
- На некоторых прошивках (Xiaomi, Huawei, Samsung) нужно разрешить
  приложению **автозапуск** и снять ограничение батареи, иначе система
  «усыпит» будильник. Добавьте приложение в исключения энергосбережения.
- **Звонки** приложение не знает — задаются вручную в настройках; меняйте
  число перед нужным днём.

## Честные ограничения

- **Только Android.** На iPhone фоновая автоматизация интерфейса запрещена
  системой — там полный автомат возможен только через облако с ботом
  (папка `../automation/`).
- **Самое хрупкое звено — авто-нажатие «отправить»** на мобильном ВК.
  Скрипт пробует кнопку и Enter; если новая вёрстка ВК их заблокирует,
  текст останется вставленным в поле — нужен один тап. Вставка текста
  надёжна. Если так случится — сообщите, подстрою селекторы в
  `ReportJs.kt` (функция `vkBootstrap`).
- Скрытый WebView и энергосбережение — самая частая причина сбоев на
  Android; поэтому важен пункт про автозапуск и батарею выше.

## Исходники

```
android/
  app/src/main/java/ru/topmts/report/
    MainActivity.kt   — экран настроек
    LoginActivity.kt  — вход в портал и ВК (сохранение cookie)
    Settings.kt       — хранение настроек
    ReportJs.kt       — JS сбора данных и отправки в WebView
    ReportRunner.kt   — прогон: портал → сбор → чат → отправка
    SendService.kt    — foreground-сервис на время отправки
    AlarmScheduler.kt — точный будильник по расписанию
    AlarmReceiver.kt / BootReceiver.kt — запуск по времени и после перезагрузки
  app/src/main/res/   — тёмный интерфейс, иконка
```
